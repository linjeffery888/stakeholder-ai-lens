----------------------------------------------------------------
Background / current role
----------------------------------------------------------------
- We run release and in-process testing: identity, potency, sterility, flow panels. Everything
  funnels through LIMS.
- Turnaround on testing is on the critical path to disposition, so any delay in QC delays a
  patient.

----------------------------------------------------------------
Pain points / friction raised (in their words)
----------------------------------------------------------------
1. Sample login gets backed up at peak, everything keyed in manually before testing can even
   start.

2. Every week I stitch together a status report from three or four different sources by hand,
   it's the same format every time and it still takes hours.

3. Out-of-spec investigations are big writeups and we do a lot of them. Documenting the
   investigation is most of the effort.

4. Stability tracking is a manual calendar and spreadsheet exercise, what's due to pull, what
   was tested, what's trending.

5. Qualifying a new assay is bench science, you run the validation experiments, there's no
   automating the wet work.

6. Reviewing certificates of analysis and transcribing results into LIMS is constant, and any
   transcription error is a deviation, so it gets double-checked, doubling the work.

7. Analysts spend hours on manual data analysis for assays, exporting from instruments,
   reformatting, and calculating in spreadsheets.

----------------------------------------------------------------
Closing note
----------------------------------------------------------------
- Skeptical that a tool can handle the GxP documentation rigor, wants to see it before
  trusting it.
